# Portfolio Summary

## Project Title

Transaction Risk Intelligence Engine – Fraud, Reconciliation and Graph Analytics

## Short Summary

Built an end-to-end financial crime and transaction risk analytics engine using Python, SQL, DuckDB, scikit-learn, NetworkX, FastAPI, Streamlit and Docker.

The project detects suspicious transactions, reconciliation breaks, risky entity clusters and operational alerts using rule-based logic, unsupervised machine learning, graph analytics and explainable case management.

## Business Value

The project simulates how financial services teams can monitor transaction risk, investigate reconciliation breaks, identify suspicious networks and prioritise analyst workload.

It demonstrates practical skills relevant to:

- financial crime analytics
- fraud detection
- AML transaction monitoring
- reconciliation operations
- risk analytics
- data quality engineering
- anomaly detection
- graph analytics
- case-management reporting
- dashboard development
- API development

## Technical Highlights

- Generated synthetic financial datasets with controlled risk patterns
- Built a DuckDB analytical database layer
- Created a reconciliation engine for missing records, duplicates and mismatches
- Added data quality validation checks
- Built rule-based transaction and customer risk scoring
- Added Isolation Forest anomaly detection
- Created plain-English alert explanations
- Used NetworkX for entity network and risky cluster analysis
- Built a unified operational case-management queue
- Exposed outputs through FastAPI
- Built an interactive Streamlit risk operations dashboard
- Added Docker and Docker Compose support

## CV Bullet Version

Built a production-style financial crime analytics engine using Python, DuckDB, SQL, scikit-learn, NetworkX, FastAPI and Streamlit to detect suspicious transactions, reconciliation breaks, risky entity clusters and operational alerts. ## Interview Explanation

This project simulates a transaction monitoring environment for financial services. I generated synthetic customer, account, merchant and transaction data, then built a pipeline that checks data quality, detects reconciliation breaks, scores transactions using transparent financial crime rules, applies machine learning anomaly detection, explains alerts in plain English and uses graph analytics to identify risky connected entities. I then combined all alerts into a case-management queue and exposed the outputs through a FastAPI service and Streamlit dashboard.

The project was designed to show both analytics and production thinking: data generation, database modelling, SQL, ML, graph analytics, explainability, API development, dashboarding, Docker and GitHub documentation.
